## 阶段流程

```mermaid
flowchart TD
  A[开始游戏] --> B[高中阶段]
  B --> B1[刷题 / 模考 / 请教 / 休息 / 社交]
  B1 --> B2[高考准备度提升]
  B2 --> B3[高中阶段结算]

  B3 --> C[本科阶段]
  C --> C1[上课 / 文献 / 实验室 / 竞赛 / 实习 / 社交]
  C1 --> C2[学业完成度与科研入门度提升]
  C2 --> C3[本科阶段结算]

  C3 --> D[硕士阶段]
  D --> D1[读文献 / 做实验 / 组会 / 写论文 / 投稿 / 会议]
  D1 --> D2[课题完成度与论文进度提升]
  D2 --> D3[硕士阶段结算]

  D3 --> E[博士阶段]
  E --> E1[深挖选题 / 长周期实验 / 论文冲刺 / 交流访问 / 指导低年级]
  E1 --> E2[博士论文与原创贡献提升]
  E2 --> E3[博士阶段结算]

  E3 --> F[青年教师阶段]
  F --> F1[申请基金 / 招募学生 / 建设平台 / 发表代表作 / 教学]
  F1 --> F2[基金积累与团队建设提升]
  F2 --> F3[青年教师阶段结算]

  F3 --> G[教授阶段]
  G --> G1[重大项目 / 培养博士生 / 原创攻关 / 学术服务 / 申报奖项]
  G1 --> G2[重大项目与人才培养提升]
  G2 --> G3[教授阶段结算]

  G3 --> H[院士候选阶段]
  H --> H1[凝练贡献 / 同行推荐 / 人才培养整理 / 学术自查 / 社会影响]
  H1 --> H2[院士评审提升]
  H2 --> I[最终结局判定]
```

## 单回合逻辑

```mermaid
flowchart TD
  A[进入回合] --> B[显示当前行动点]
  B --> C[玩家选择一个可执行行动]
  C --> D[扣除行动点]
  D --> E[结算属性变化]
  E --> F[结算阶段进度变化]
  F --> G{剩余行动点还能执行任一行动吗}
  G -- 能 --> C
  G -- 不能 --> H[自动回合结算]
  H --> I{是否触发随机事件}
  I -- 是 --> J[显示事件选择]
  J --> K[玩家选择事件选项]
  K --> L[结算事件效果]
  L --> M{阶段是否结束}
  I -- 否 --> M
  M -- 否 --> N[进入下一回合]
  M -- 是 --> O[阶段结算]
  O --> P{是否最后阶段}
  P -- 否 --> Q[进入下一阶段]
  P -- 是 --> R[结局判定]
```

## 结局判定

```mermaid
flowchart TD
  A[院士候选阶段结束] --> B{当选院士?}
  B -- 是 --> B1[当选院士]
  B -- 否 --> C{学术大师?}
  C -- 是 --> C1[学术大师]
  C -- 否 --> D{优秀教授?}
  D -- 是 --> D1[优秀教授]
  D -- 否 --> E{转向产业研发?}
  E -- 是 --> E1[转向产业研发]
  E -- 否 --> F{科研受挫?}
  F -- 是 --> F1[科研受挫]
  F -- 否 --> G[平凡但完整]
```

## 读图顺序

1. 先看“总览”，理解游戏由目标、核心循环、属性和阶段进度组成。
2. 再看“阶段流程”，理解玩家从高中一路走到院士候选。
3. 然后看“单回合逻辑”，理解为什么现在没有手动结束回合。
4. 最后看“结局判定”和“主要数据文件”，方便继续开发和调数值。
