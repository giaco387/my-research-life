# 《科研之路》游戏开发文档

## 1. 项目概述

《科研之路》是一款以学生成长和科研职业发展为主题的单机模拟经营游戏。玩家从高中阶段开始，通过学习、考试、选择专业、进入实验室、发表论文、申请项目、建立学术声望等方式，逐步经历本科、硕士、博士、博士后/青年教师等阶段，最终目标是成为院士。

游戏强调长期规划、资源取舍和随机事件带来的不确定性。玩家需要在知识积累、身体健康、心理压力、人际关系、科研产出和学术声望之间做平衡，而不是单纯堆叠某一个数值。

## 2. 目标平台与形式

- 首版平台：Web 单机版
- 推荐技术：React + Vite
- 游戏类型：回合制模拟经营 / 文字事件 / 轻策略养成
- 操作方式：鼠标点击为主
- 单局时长：首版 20-40 分钟，完整版 1-2 小时
- 存档方式：浏览器 localStorage，后续可扩展为云存档

## 3. 核心体验

玩家每一回合会获得有限行动点，并在若干行动中选择本回合的安排。例如学习、休息、社交、进实验室、写论文、申请基金、参加会议等。每个行动会改变角色属性、积累阶段进度，并可能触发事件。

核心乐趣来自三类取舍：

- 短期收益与长期发展：刷题、肝实验、赶论文可以快速提升成果，但会增加压力和损耗健康。
- 稳定路线与高风险机会：竞赛、顶会投稿、重大项目、跨学科方向有更高回报，也更容易失败。
- 个人能力与外部环境：导师、同门、课题组、学校平台、政策窗口会影响发展路径。

## 4. 游戏目标

### 4.1 终极目标

成为院士。

### 4.2 阶段目标

- 高中：考入理想大学，形成基础能力。
- 本科：选择专业方向，进入实验室，争取保研/考研/直博机会。
- 硕士：完成课题训练，发表论文，决定就业、读博或转向。
- 博士：形成稳定研究方向，产出高质量成果，完成答辩。
- 博士后/青年教师：申请基金，建立团队，扩大学术影响力。
- 教授/学术带头人：承担重大项目，培养学生，获得重要奖项。
- 院士候选：积累原创贡献、学术声望、社会影响，通过评审。

## 5. 玩家属性

### 5.1 基础属性

| 属性 | 含义 | 主要影响 |
| --- | --- | --- |
| 知识 | 学科基础和学习能力 | 考试、论文质量、课题理解 |
| 毅力 | 面对长期压力的持续投入能力 | 高强度行动收益、失败后的恢复 |
| 专注 | 本回合行动效率 | 学习、写作、实验成功率 |
| 体力 | 身体状态 | 行动点恢复、过劳风险 |
| 压力 | 心理负担 | 过高会触发负面事件 |
| 情商 | 沟通、合作、处理关系能力 | 导师关系、团队合作、会议交流 |
| 资金 | 可支配资源 | 购买设备、参加会议、维持生活 |
| 声望 | 外界认可 | 升学、基金、职位、院士评审 |

### 5.2 科研属性

| 属性 | 含义 | 主要影响 |
| --- | --- | --- |
| 文献理解 | 阅读和吸收论文的能力 | 开题、选题、论文写作 |
| 实验/工程能力 | 做实验或构建系统的能力 | 课题推进、数据质量 |
| 写作能力 | 论文和申请书表达能力 | 投稿、基金申请 |
| 创新性 | 发现新问题和提出新方法的能力 | 高水平成果、原创贡献 |
| 合作网络 | 与导师、同门、同行的关系 | 合作论文、推荐、资源获取 |
| 学术贡献 | 游戏后期核心评分 | 院士评审的关键门槛 |

## 6. 核心资源

### 6.1 行动点

每回合获得固定行动点，代表时间和精力。不同阶段行动点上限不同：

- 高中：每回合 6 点
- 本科：每回合 7 点
- 硕士：每回合 8 点
- 博士及以后：每回合 9 点

体力过低时，下回合行动点减少。压力过高时，行动可能失败或触发负面事件。

### 6.2 阶段进度

每个阶段有独立进度条，例如：

- 高中：高考准备度
- 本科：学业完成度、科研入门度
- 硕士：课题完成度、论文进度
- 博士：博士论文进度、原创贡献
- 教师阶段：团队建设、项目积累、学术影响

达到阶段目标后进入阶段结算，根据属性和成果决定下一阶段起点。

## 7. 阶段设计

### 7.1 高中阶段

目标：高考成功，进入大学。

主要行动：

- 刷题训练：提升知识，增加压力。
- 模拟考试：提升考试表现，可能暴露短板。
- 请教老师：提升知识和专注，消耗行动点。
- 休息调整：恢复体力，降低压力。
- 运动放松：提升体力和毅力，降低压力。
- 与同学交流：提升情商，可能获得事件帮助。

关键事件：

- 月考失利
- 家长施压
- 志愿填报
- 临考失眠
- 朋友邀约

阶段结算：

根据知识、专注、压力和随机波动计算高考结果。结果影响大学层级和本科初始资源。

### 7.2 本科阶段

目标：确定方向，获得升学机会。

主要行动：

- 认真上课：提升知识和绩点。
- 阅读文献：提升文献理解。
- 进入实验室：提升科研属性，增加压力。
- 参加竞赛：提升声望和毅力，高风险高收益。
- 社团/社交：提升情商和合作网络。
- 企业实习：提升资金和现实认知，可能分散科研投入。
- 联系导师：解锁科研机会。

关键事件：

- 导师邀请进组
- 同学提前发表论文
- 课程挂科危机
- 保研名额竞争
- 专业兴趣动摇

阶段结算：

玩家可进入硕士、直博、就业结局或重开学术路线。首版建议只实现硕士/直博两条主线。

### 7.3 硕士阶段

目标：完成科研训练，产出第一批成果。

主要行动：

- 读文献：提升选题能力。
- 做实验/跑模型：推进课题进度。
- 组会汇报：提升表达和导师关系。
- 写论文：推进论文进度。
- 投稿：根据论文质量判定接收、修改或拒稿。
- 参加会议：提升声望和合作网络。
- 调整方向：重置部分进度，但提升未来潜力。

关键事件：

- 课题做不出来
- 导师临时换方向
- 审稿意见大修
- 同门合作冲突
- 找工作还是读博

阶段结算：

根据论文、导师关系、知识和压力判断是否顺利毕业，以及是否获得读博机会。

### 7.4 博士阶段

目标：形成独立研究方向，完成博士论文。

主要行动：

- 深入选题：提升创新性和原创贡献。
- 长周期实验：高投入，高产出，高失败风险。
- 论文写作：推进成果发表。
- 申请交流访问：提升合作网络和声望。
- 指导低年级学生：提升管理能力。
- 心理调适：降低压力，维持长期稳定。
- 答辩准备：推进毕业进度。

关键事件：

- 关键实验失败
- 顶会/顶刊拒稿
- 导师资源不足
- 国际合作机会
- 延毕风险
- 论文被竞争团队抢发

阶段结算：

博士毕业结果由博士论文进度、代表作质量、原创贡献、健康状态共同决定。毕业后进入博士后/青年教师阶段。

### 7.5 博士后/青年教师阶段

目标：从个人科研转向独立 PI。

主要行动：

- 申请青年基金
- 招募学生
- 建设实验平台
- 发表代表作
- 参加学术会议
- 拓展合作
- 平衡教学与科研

关键事件：

- 基金申请失败
- 学生培养问题
- 学校考核压力
- 设备经费不足
- 获得重要合作机会

阶段结算：

根据基金、论文、团队、声望判断是否获得稳定职位或晋升。

### 7.6 教授/学术带头人阶段

目标：形成学派和重要原创贡献。

主要行动：

- 组织重大项目
- 培养博士生
- 建设研究团队
- 发表高影响成果
- 参与学术共同体服务
- 推动成果转化
- 申请国家级奖项

关键事件：

- 团队成员离开
- 重大项目评审
- 学术争议
- 成果转化机会
- 学生取得突破

阶段结算：

积累足够原创贡献、声望、奖项和人才培养成果后，解锁院士候选阶段。

### 7.7 院士候选阶段

目标：通过院士评审。

主要判定维度：

- 原创贡献
- 代表性成果
- 学术声望
- 人才培养
- 学术道德
- 社会影响
- 年龄与健康状态

结局：

- 当选院士：最高成功结局。
- 学术大师：未当选但贡献卓越。
- 优秀教授：稳定成功结局。
- 转向产业/管理：分支结局。
- 身心耗竭：失败或警示结局。

## 8. 行动系统

每个行动包含以下字段：

```ts
type Action = {
  id: string;
  stage: StageId;
  name: string;
  cost: number;
  description: string;
  effects: Partial<PlayerStats>;
  progressEffects?: Partial<StageProgress>;
  requirements?: Requirement[];
  risk?: RiskConfig;
};
```

行动设计原则：

- 每个行动都应有代价，不提供无脑最优选项。
- 高收益行动通常伴随压力、体力损耗或失败风险。
- 休息类行动必须有真实价值，否则玩家会倾向一直压榨角色。
- 后期行动要从“提升个人属性”逐渐转向“管理团队和资源”。

## 9. 事件系统

事件分为四类：

- 阶段事件：只在特定阶段出现。
- 属性事件：当压力、体力、声望等达到阈值时触发。
- 随机事件：用于增加变化和叙事。
- 里程碑事件：阶段转换、首次发表、基金获批、晋升等。

事件结构：

```ts
type Event = {
  id: string;
  stage?: StageId;
  title: string;
  description: string;
  trigger: EventTrigger;
  choices: EventChoice[];
};

type EventChoice = {
  label: string;
  effects: Partial<PlayerStats>;
  progressEffects?: Partial<StageProgress>;
  nextEventId?: string;
};
```

事件设计原则：

- 避免把科研简化为纯粹吃苦。游戏应呈现选择、环境、合作和运气。
- 事件选项不应只有明显正确答案。
- 重要事件可以延迟反馈，例如一次换方向会在数回合后体现收益。
- 学术道德事件必须明确：造假、剽窃、数据篡改等行为即使短期有收益，也应带来长期严重后果。

## 10. 成果系统

### 10.1 论文

论文由多个数值决定：

- 选题价值
- 数据/实验质量
- 写作质量
- 创新性
- 投稿期刊/会议等级
- 审稿随机性

投稿结果：

- 接收
- 小修
- 大修
- 拒稿

拒稿不应等于完全失败，可以保留一部分进度并增加经验。

### 10.2 基金

基金申请由以下因素影响：

- 申请书质量
- 前期成果
- 学术声望
- 方向热度
- 合作网络
- 随机评审波动

基金带来的收益：

- 资金增加
- 团队容量增加
- 声望增加
- 解锁大型项目行动

### 10.3 奖项

奖项是后期的重要里程碑，包括：

- 优秀论文奖
- 青年人才项目
- 国家级科研奖项
- 国际学会奖项

奖项主要用于推动声望和院士评审。

## 11. 数值框架

### 11.1 属性范围

首版建议所有基础属性使用 0-100：

- 0-20：危险或薄弱
- 21-50：普通
- 51-75：良好
- 76-90：优秀
- 91-100：顶尖

压力是反向属性：

- 0-40：健康
- 41-70：紧张
- 71-90：危险
- 91-100：崩溃风险

### 11.2 风险判定

示例：

```ts
successRate =
  baseRate
  + knowledge * 0.2
  + focus * 0.15
  + relatedSkill * 0.25
  + reputation * 0.1
  - pressure * 0.15
  - difficulty;
```

最终成功率应限制在 5%-95% 之间，避免完全确定。

### 11.3 健康与压力惩罚

- 体力低于 20：下回合行动点 -2。
- 压力高于 80：行动成功率下降，并可能触发焦虑、失眠、效率低下事件。
- 压力达到 100：强制进入恢复回合，损失阶段进度或声望。

## 12. UI 设计

### 12.1 主界面

主界面包含：

- 当前阶段、年份、回合数
- 玩家属性面板
- 阶段目标和进度条
- 行动卡片区
- 当前事件弹窗
- 历史日志

### 12.2 交互流程

单回合流程：

1. 显示当前状态。
2. 玩家选择多个行动，直到行动点耗尽或主动结束回合。
3. 结算行动效果。
4. 判定是否触发事件。
5. 更新阶段进度。
6. 若达到阶段结算条件，进入阶段总结。

### 12.3 视觉方向

建议采用清晰、克制的校园与科研工作台风格：

- 高中：教室、试卷、倒计时、操场。
- 本科：校园、图书馆、实验室入口。
- 硕博：实验台、论文、代码、组会白板。
- 教师阶段：办公室、会议、团队、项目材料。

整体不建议做成过度卡通的爽游风格。主题可以轻松幽默，但核心表达应尊重科研成长的复杂性。

## 13. MVP 范围

首个可玩版本建议只实现：

- 高中、本科、硕士三个阶段
- 8 个基础属性
- 每阶段 6-8 个行动
- 每阶段 5-8 个事件
- 简单阶段结算
- 3 个结局：顺利读博、转向就业、科研受挫
- localStorage 存档
- 历史日志

MVP 不建议首版实现：

- 完整院士评审
- 复杂导师 AI
- 复杂论文审稿模拟
- 多角色团队管理
- 联网排行榜

这些可以放到后续版本。

## 17. 设计风险

- 数值过重会变成表格游戏，需要事件和叙事增加代入感。
- 随机性过强会让玩家觉得努力无效，需要保留可控策略空间。
- 主题容易变成单纯“卷”的模拟，需要让休息、合作、选择方向同样有价值。
- 院士目标跨度很长，必须通过阶段性目标维持节奏。
- 学术现实复杂，游戏表达应避免把科研成功简化为单一努力或单一关系。

## 18. 需要进一步确认的问题

后续开发前建议确认：

- 游戏是偏严肃现实，还是偏轻松讽刺？
- 是否允许失败后继续转向其他职业结局？
- 学科方向是否区分，例如数学、计算机、生物、材料、医学？
- 是否加入导师性格和课题组文化系统？
- 美术是纯文字卡片、像素风、校园插画风，还是管理面板风？
- 单局是否要追求真实漫长，还是更偏快节奏重复游玩？
